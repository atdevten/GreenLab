KEY DESIGN PRINCIPLES — AI + FINE-TUNING
LoRA / Adapter Fine-tuning
Users fine-tune only adapter layers, not full model weights. 100x cheaper, 10x faster. Base model shared across all users.
Predictions as First-Class Fields
Predicted values stored as channel fields in InfluxDB alongside real readings. Same API to query both.
Sensor + Video Multimodal
Fine-tuning UI lets users select sensor channels AND video clips as joint training inputs. Fusion model handles both.
Drift Detection & Auto-Retrain
Evidently monitors prediction accuracy in production. Triggers notification (or auto job) when data distribution shifts.
Spot GPU for Training Jobs
Fine-tuning jobs use spot/preemptible GPU instances. Checkpointing every epoch. 80% cost reduction vs on-demand.
Model Marketplace
Users can publish fine-tuned models to a shared marketplace. Others import and further specialize them for their channel.
